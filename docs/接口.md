# 接口与数据约定（与当前代码对齐）

官方原始 XML 位于项目根目录 **`data/`**（`Restaurants_Train_v2.xml`、`Restaurants_Test_Gold.xml`、`Laptop_Train_v2.xml`、`Laptops_Test_Gold.xml`）。**请勿在 `ate/` 下再放重复的训练/测试 XML。**

## 1. 数据划分（`pipeline/semeval_data.py` + prepare 脚本）

- ATE 与 ASC 共用同一套 **`sentence_id`** 上的 train/val 划分（`VAL_RATIO=0.1`，`SEED=42`）。
- **Test** 固定为整份 **Gold Test** XML（带 `aspectTerm` 极性）。
- `ate_prepare_data.py` / `asc_prepare_data.py` 支持 `--domain restaurant|laptop`，输出分别为 `ate_data_restaurant` / `asc_data_restaurant` 与 `ate_data_laptop` / `asc_data_laptop`。

## 2. 端到端 Pipeline（`pipeline/run_pipeline.py`）

- 写出 **`ate_predictions_{domain}_{model}.jsonl`** 与 **`e2e_predictions_{domain}_{model}.jsonl`**（JSON Lines，含 `sentence_id`、span、`ate_confidence`、情感标签等）。
- ASC 侧批量接口见 **`pipeline/asc_adapter.py`**：`predict_sentiment` / `predict_sentiment_batch` → `"positive"` / `"negative"` / `"neutral"`。

## 3. 独立脚本 `ate/ate_predict.py`（遗留）

- 仍为 **JSON 数组**、每 aspect 一条记录，**不等价**于 pipeline 的 JSONL 规范；仅用于单独调试 ATE。正式实验请用 **`python -m pipeline.run_pipeline`**。

## 4. ASC 评估脚本 `asc/asc_evaluate.py`

- 面向 **HuggingFace Dataset** 的批量 `predict`；与 pipeline 包装层分工不同，行为一致即可。

## 5. Pipeline 配置路径（`pipeline/config.py`）

- `RAW_DATA` 中 `train_xml` / `test_xml` 已指向 **`data/`** 下 v2 Train 与 Gold Test，与 prepare 脚本一致。
